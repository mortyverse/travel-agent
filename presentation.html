Travel

Problem

여행 계획, 왜 어려울까요?

시간 소모
코스, 숙소, 교통을 각각 따로 검색하고 조합하는 데 수 시간이 필요합니다.
비효율적 동선
지리적 위치를 고려하지 않은 일정은 불필요한 이동 시간을 만들어냅니다.
정보 파편화
코스·숙소·교통·비용 정보가 흩어져 있어 통합된 일정표를 만들기 어렵습니다.
핵심 질문: 사용자가 여행 조건만 말하면, AI가 완성형 일정표를 자동으로 만들어줄 수 있을까?

Agents

각 에이전트의 역할

Orchestrator
오케스트레이터
6가지 조건을 순서대로 수집하고 3개 에이전트에 병렬 배분
Course
코스 플래너
15분 근접 규칙 기반 동선 최적화 및 목적별 장소 큐레이션
Accommodation
숙소 플래너
예산·동선 중심부 기준 숙소 2종 이상 추천 (1순위 + 대안)
Transport
교통 플래너
장거리 이동 + 현지 교통 경로 및 구간별 비용 계산
Editor
편집 플래너
3개 결과를 통합해 DAY별 표 + 비용 요약 + 팁 포함 최종 출력

Input Conditions

오케스트레이터가 수집하는 6가지 조건

01
여행지 어느 지역으로 여행하는지 (예: 제주, 경주, 강릉)
02
여행 기간 몇 박 며칠인지 (예: 1박 2일, 2박 3일)
03
여행 목적 자연, 맛집, 역사, 힐링, 액티비티 등
04
이동 수단 대중교통 또는 자가용
05
예산 수준 저가 / 중간 / 고급
06
동행 인원 혼자 / 커플 / 가족 / 친구 몇 명
수집 완료 시 → 코스·숙소·교통 플래너 동시 실행 → 편집 플래너가 통합 → 완성형 일정표 반환

Key Feature

플래닝 오버레이 애니메이션

에이전트가 분석 중일 때 사용자가 막연히 기다리는 문제를 해결하기 위해, 각 에이전트의 작업 진행 상황을 실시간으로 보여주는 오버레이를 구현했습니다.

터미널 스타일 로그각 에이전트 작업 단계를 한 줄씩 순차 출력
병렬 시각화3개 카드가 시차를 두고 동시에 진행
완료 피드백각 에이전트 완료 시 초록 체크 + 카드 테두리 변경
진행 바전체 진행률을 상단 프로그레스 바로 표시
